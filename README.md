**TDFinancialHealth**

*Overview*

Developed a set of dynamic web pages that provided informative insights about TD's upcoming financial health services platform — including a signup form and a page displaying its results.

*Solution Approach*

1. Robust Navigation & Information Architecture 📚
- Developed a clean, responsive navigation bar and footer.
- Ensured consistent branding aligned with TD design guidelines.
- Built flexible SCSS structures using BEM methodology to support future expansion.

2. Accessible & Secure Signup Experience 🛡️
- Built a semantic, accessible HTML form for beta program enrollment.
- Added enhanced security considerations, such as proper form validation and consent collection.
- Designed a consistent visual experience for both the signup and confirmation pages.

3. API-Integrated Submission Workflow 🔗
- Implemented JavaScript logic to send and retrieve form submissions from the backend API.
- Automatically populated the signups table with user entries for smoother internal review.
- Ensured failure states and success responses were handled gracefully with specific error handling pages.

4. Responsive & Inclusive Design Principles 🌐
- Applied TD branding assets and type scales.
- Ensured layouts adapt cleanly across desktop, tablet, and mobile.
- Considered accessibility best practices, including contrast, touch targets, and screen-reader compatibility.

*Additional Features*

1. AI ChatBot 🤖
- Early Data Scrape: Allows TD to scrape early user concerns to address in product features.

2. Form Validations ✅
- Data Quality (characters): eliminates data discrepancies & ensure successful signups.
- Duplication Limitation: affirms users’ original submissions & reduces access issues with user profiles for future program roll out.

3. Form Security Features 🔒
- Trust & Transparency: Beta website users gain early confidence with TD products.

4. Enhanced Navigation 🧭
- Smooth UX: A more modern & user-friendly approach and reduced bounce rates between pages.

5. API Error Handling ⚠️
- Specific Messaging: Provides users with error rationale to reduce confusion and also service requests.

6. Responsive Design & Accessibility 🎨
- Device Friendly: All content is clear on a range of resolutions.
